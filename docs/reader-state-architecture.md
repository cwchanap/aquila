# Reader state architecture

- **`readerState` (`src/lib/reader-state.svelte.ts`)** is the single canonical reactive owner of reader progression (`storyId`, `currentSceneId`, `locale`, `dialogueIndex`), the derived scene payload (`dialogue`, `choice`, `canGoNext`), and asynchronous presentation state (`loadStatus`, `loadError`, `hasActivePayload`, `activeFlow`). Only progression is serialized to the URL and localStorage. Loaded dialogue, choices, flow, and load state are runtime-only.
- **`reader-intent.ts`** performs the lightweight half of restoration. `selectReaderIntent` chooses a registered story from the URL, persisted state, or the configured default without importing story content. Once the asynchronous payload arrives, `validateLoadedIntent` resolves its scene/index or returns the explicit fallback/soft-reject policy. Initial stale persisted state may fall back to the default; invalid known-story `popstate` destinations retain the active session and reconverge the URL.
- **`reader-session.ts`** owns pure session mechanics: serialization, migration, index clamping, and flow-based scene validation. It does not load story modules.
- **`@aquila/stories/async`** is the story payload boundary. `loadStoryContent(storyId, locale)` dynamically imports one registered story, combines dialogue/choices/flow, and caches successful promises by normalized `storyId:locale`. Rejected loads are evicted so a later document load or retry can try again.
- **`ReaderManager`** is the plain-TypeScript orchestrator. It mounts the shell before loading, owns the active payload and pending intent, and uses a generation token so late initial/replacement results cannot apply. A successful load atomically updates progression, scene payload, `activeFlow`, and ready state before URL/persistence sync. A replacement load or error keeps the prior active payload mounted beneath an inert loading/error overlay. `popstate` cancels pending line replacement; an active-session destination without `story` is soft-rejected without loading, invalidates pending replacement work, and canonicalizes to the active session. `pagehide` flushes URL/persistence. `destroy()` invalidates loads, removes listeners/timers, and unmounts for test isolation or future SPA teardown.
- **`ReaderShell`** is the stable reactive store-to-props bridge. It renders standalone loading/error surfaces before the first payload, preserves the reader while replacements load or fail, and forwards `activeFlow` to both desktop and mobile readers.
- **`NovelReader` / `MobileNovelReader`** are pure controlled components: data in via props, `onIndexChange`/actions out, no store import. Visible history derives from the index; the typewriter follows the two-signal algorithm (scene-ref change animates the first line; index change animates only if self-initiated, else snaps).
- **URL contract:** `story` / `scene` / `dialogue=N` (1-based). `locale` is path-owned, never in the URL.
