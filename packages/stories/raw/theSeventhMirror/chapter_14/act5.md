**旁白**：14:20。都政資料閱覽室。

**旁白**：澪第二次走進來。上午的閱覽申請還在櫃台。她出示身分證件。填寫追加調閱單。報上同一個資料包編號。

**資料室職員**：追加調閱——運用結果速報及復舊確認摘要。同一資料包內。請在閱覽室等候。閉館前三十分鐘為拍攝截止時間。

**朝倉澪**：好。

**旁白**：她坐到上午同一個位置。綠罩燈。木桌。安靜。幾分鐘後，職員推著資料車過來。一個新的文件夾。比上午的厚。

**旁白**：她打開文件夾。翻到目錄頁。

**旁白**：目錄上列著附件清單。她的目光逐條往下掃。

**旁白**：事前運用變更申請——文件 A。上午已經看過。

**旁白**：多通道測試參數附頁——文件 B。上午已經看過。

**旁白**：事後運用結果速報／復舊確認摘要——文件 C。

**朝倉澪** [determined]：（內心）⋯⋯在這裡。文件 C。事後。運用結果。復舊確認。

**旁白**：她翻到文件 C 的頁面。

**旁白**：文件格式和文件 A、B 不同。不是 TKS 的制式表格。是灣岸新交通的內部報告格式。抬頭寫著——灣岸新交通運用部。值班主管署名欄。下方有安全委員會收件戳。

**朝倉澪** [determined]：（內心）⋯⋯這份文件不是 TKS 寫的。是灣岸新交通的值班主管寫的。送交安全委員會。和文件 A、B 的來源不同。文件 A 是 TKS 提出、交通公司核准。文件 B 是 TKS 技術部製作、交通公司測試負責人確認。文件 C 是交通公司自己形成的事後報告。三份文件。三個不同的人簽署。

**旁白**：她往下看。文件 C 的正文。

**旁白**：對應申請——文件 A 的變更申請編號。同一個編號。和上午看到的一致。

**旁白**：參照參數——文件 B 的附頁版本 B-2。和她上午拍攝的參數附頁版本一致。

**朝倉澪** [determined]：（內心）⋯⋯同一個申請編號。同一個附頁版本。文件 C 明確引用了文件 A 和文件 B。它不是獨立文件。它是事後報告，記錄前兩份文件的計畫是否被執行。

**旁白**：她繼續往下看。

**旁白**：啟用時間——星期一 20:55。

**旁白**：測試完成——21:35。

**旁白**：復舊完成——21:36。

**朝倉澪** [determined]：（內心）⋯⋯20:55 啟用。21:35 完成。21:36 復舊。文件 A 預定的測試窗是 20:55 到 21:35。文件 C 記錄的實際時間和預定一致。一分鐘內完成復舊。配置確實啟用了。不是只有計畫。是確實執行了。

**旁白**：再往下。

**旁白**：配置結果——按核准映射啟用。伺服器端未記錄設定偏差。

**朝倉澪** [determined]：（內心）⋯⋯按核准映射啟用。意思是——文件 A 計畫把 TOKYO 暫接 G07，文件 C 確認這個配置確實被啟用了。伺服器端未記錄設定偏差——意思是文件 B 的參數，包括 +7000ms，按核准值發送。沒有偏差。

**旁白**：再往下。

**旁白**：復舊結果——回復標準運用群。公開版遮蔽實際群號。

**朝倉澪** [determined]：（內心）⋯⋯回復了。21:36 回復標準運用群。TOKYO 不再接 G07。回到原本的群。公開版不寫出原本的群號——和文件 A 一樣，標準運用群被遮蔽。但 G07 是臨時測試群。測試結束就還回去了。

**旁白**：她把文件 C 的每一欄抄進筆記本。一欄一欄抄。字寫得很慢。

**旁白**：抄完之後，她翻到前面。文件 A、B、C 的抄錄並排放在一起。

**旁白**：文件 A。事前。TKS 提出。交通公司核准。計畫將 TOKYO 暫接 G07。測試窗 20:55 到 21:35。

**旁白**：文件 B。事前。TKS 技術部製作。交通公司測試負責人確認。手機推送 +7000ms。同一申請編號。附頁版本 B-2。

**旁白**：文件 C。事後。交通公司值班主管製作。安全委員會收件。確認 20:55 啟用。21:36 復舊。按核准參數執行。未記錄偏差。

**朝倉澪** [determined]：（內心）⋯⋯計畫。參數。執行。回復。三份文件。三個不同的人簽署。同一個申請編號把它們串起來。文件 A 只證明了計畫。文件 B 只證明了參數。文件 C 證明了——計畫被執行了。參數被用了。配置被啟用了。然後被回復了。這才是完整的鏈。

**旁白**：她看著這三份抄錄。看了很久。

**朝倉澪** [determined]：（內心）⋯⋯佐伯說——事前文件不等於實際執行。他說得對。文件 A 和 B 是事前。它們不能證明配置真的啟用了。但文件 C 是事後。它記錄了啟用時間、完成時間、復舊時間。它引用了文件 A 的申請編號和文件 B 的附頁版本。它由交通公司值班主管簽署，不是 TKS 自己。所以這不是 TKS 的單方面說法。是交通公司確認 TKS 的計畫被執行了。

**旁白**：她翻到文件 C 的附件目錄頁。

**旁白**：目錄下方列著幾個附件名稱。她的目光逐條往下。

**旁白**：別紙 A——運用變更申請正本。文件 A。

**旁白**：別紙 B——多通道測試參數附頁。文件 B。

**旁白**：別紙 C——設定檔別名對照表。

**旁白**：她的手指停在那一行。

**朝倉澪** [determined]：（內心）⋯⋯別紙 C。設定檔別名對照表。

**旁白**：她往右看。別紙 C 那一行後面有幾個欄位。

**旁白**：附件名稱——設定檔別名對照表。

**旁白**：頁數——未載。

**旁白**：主管部門——TKS 系統管理部。

**旁白**：原始承包交付編號——列有一串編號。

**旁白**：閱覽狀態——基於系統安全不提供公開閱覽。

**朝倉澪** [determined]：（內心）⋯⋯不提供公開閱覽。從一開始就不提供。不是今天才被遮掉。不是因為我開始調查才被刪除。是這份附件本身的閱覽狀態就是——不公開。它和文件 A、B、C 在同一個資料包裡。A、B、C 可以看。別紙 C 不能看。

**旁白**：她看著那行字。基於系統安全不提供公開閱覽。旁邊有一張替代頁。替代頁上只有附件名稱、主管部門、交付編號。沒有內容。

**朝倉澪** [determined]：（內心）⋯⋯設定檔別名對照表。它記錄的是——設定檔的名稱和別名之間的對應。TOKYO 可能有一個或多個別名。舊版本的名稱。測試用短代碼。或者——其他什麼。我不知道。因為它不公開。但我知道它存在。我知道它的主管部門是 TKS 系統管理部。我知道它的原始承包交付編號。這些都是公開的目錄資訊。

**旁白**：她把別紙 C 的目錄資訊抄進筆記本。附件名稱。主管部門。交付編號。閱覽狀態。

**旁白**：然後她在旁邊寫——

**旁白**：別紙 C：存在。不公開。不是事後刪除。從最初便不公開。

**朝倉澪** [determined]：（內心）⋯⋯這是下一步的入口。別紙 C 被哪些文件引用。它的交付編號能追到什麼地方。千田有沒有參與它的維護或交付。這些都是 Chapter 15 的事。今天——我只需要確認它存在。確認它從一開始就不公開。確認它不是因為我查到才消失的。

**旁白**：她翻回文件 C 的正文。再看一遍。

**旁白**：然後她注意到文件 C 的說明欄裡有一段備註。字很小。印在頁面底部。

**旁白**：備註——本資料包中可公開項目包括臨時測試群號、測試時段、通道順序補正及復舊確認。群內端點、實體位置、維護帳號及設定檔別名對照表屬系統安全資訊，不提供公開閱覽。

**朝倉澪** [determined]：（內心）⋯⋯這就是為什麼 G07 可以公開。它是一個臨時空殼群號。不含端點。不含設備。不含帳號。只是一個編號。G07。第七群。任何人看到 G07 都不能從中定位任何設備或個人。所以它可以寫在公開文件裡。

**朝倉澪** [determined]：（內心）⋯⋯但 Chapter 13 那份摘要表——把同步群和端點範圍寫在同一格。端點範圍是敏感的。所以整格被遮蔽。這份文件 A 把臨時群號單獨列出，不含端點。所以可以公開。兩種處理方式不同。不矛盾。一個是合併欄位整格遮蔽。一個是分開欄位只遮蔽敏感部分。

**旁白**：她拿起手機。打開和佐伯的對話。把文件 C 的去識別化裁切圖傳過去。只裁切了啟用時間、復舊時間和配置結果。沒有露出文件編號。沒有露出簽署人姓名。

**旁白**：幾分鐘後，佐伯回了。

**佐伯達也**：事後文件。好。這補了執行證據的缺口。但——群號可在不同部署內各自編排。研究資料專案和灣岸警報專案都出現 G07，仍可能只是普通重號。不能因為兩邊都有 G07 就當成同一群。

**朝倉澪** [determined]：（內心）⋯⋯佐伯說得對。文件 C 證明了星期一 TOKYO 確實接上 G07。但它不能證明——這個 G07 就是悠真預約頁上那個 G07。兩條業務線可能各自擁有一個第七群。群號一樣。內容不同。就像兩棟大樓都有七樓。七樓和七樓不等於同一層。

**旁白**：她回覆佐伯。

**訊息**：知道。不畫等號。文件只證明星期一 TOKYO 用過 G07。不證明研究 G07 和警報 G07 是同一批端點。

**旁白**：佐伯回了最後一條。

**佐伯達也**：可以記錄。不要解讀。
